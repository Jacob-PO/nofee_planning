배경 및 목적

현재 상황

현재 노피 어드민에서는 두 가지 콘텐츠 유형만 지원하고 있습니다:

  • 이벤트 피드: 하나의 상품을 지정하여 이벤트 가격을 작성, 상품을 한 개씩 업로드
  • 일반 피드: 일반 글과 일반 이미지를 업로드

문제점

  • 판매점이 여러 상품을 등록하려면 반복적으로 하나씩 업로드해야 함
  • 대량의 상품 정보를 효율적으로 관리하기 어려움
  • 가격 변동 시 일일이 수정해야 하는 번거로움
  • 고객이 매장의 전체 상품 라인업을 한눈에 보기 어려움

해결 방안

시세표 콘텐츠 유형을 추가하여 판매점이 표 형식으로 대량의 상품을 쉽게 업로드하고 관리할 수 있도록 합니다. 시세표는 동네성지의 각 스토어 메인에 고정 노출되어 고객이 매장의 모든 상품을 한눈에 확인할 수 있습니다.

주요 기능

1. 시세표 입력 및 다운로드 기능
  • 표 형식으로 여러 상품을 한 번에 입력
  • 상품명, 용량, 월요금, 통신사별 가격을 행 단위로 입력
  • 가격 입력 방식: 만원(만₩) 단위로 입력 (예: 55만원 → "55" 입력, 60만원 → "60" 입력)
  • 고객 화면 표시: 입력한 만원 단위 가격이 그대로 표시됨 (예: "55" 입력 → "55만" 표시)
  • 행 추가/삭제: 버튼으로 상품 행 추가/제거
  • 임시저장: 작업 중간에 저장 가능 (30초마다 자동저장)
  • 직관적인 인라인 편집 (셀을 클릭하여 바로 수정)
  • 이미지 다운로드: 완성된 시세표를 노피 디자인 이미지로 저장
💡 가격 입력 및 표시 규칙
  • 어드민 입력: 만원 단위 정수로 입력 (양수 또는 음수 가능)
    • 양수 예시: 55만원 → "55", 60만원 → "60"
    • 음수 예시: 12만원 페이백 → "-12", 5만원 페이백 → "-5"
  • 소수점 입력 불가: 0.5와 같은 소수점 값은 입력할 수 없습니다 (정수만 허용)
  • 음수 가격 의미: 음수는 고객이 돈을 받고 상품을 구매하는 "페이백" 또는 "리베이트"를 의미합니다
    • 예: -12만원 = 12만원을 돌려받고 상품 구매
  • DB 저장: 입력한 값에 10,000을 곱하여 원 단위로 저장 (55 → 550,000원, -12 → -120,000원)
  • 고객 표시:
    • 양수: "55만", "60만" (일반 가격)
    • 음수: "12만원 페이백", "5만원 페이백" (페이백 표시)
  • 공란 처리: 가격을 입력하지 않으면 고객 화면에서 "-"로 표시되며 선택 불가

3. 모바일 고객 화면
  • 탭 구조 개편: 홈, 시세표, 후기 3개 탭으로 구성
  • 홈 탭: 기존 소식 탭의 기능을 흡수하여 "우리매장 소식" 섹션으로 통합
  • 시세표 탭: 전용 시세표 화면으로 분리하여 집중도 향상
  • 모바일 최적화된 카드형 레이아웃
  • 용량 선택 드롭다운: 동일 기기의 여러 용량 옵션 선택 가능 (단일 옵션인 경우 드롭다운 미표시)
  • 요금제 선택 드롭다운: 월요금 옵션 선택 가능 (단일 옵션인 경우 드롭다운 미표시)
  • 실시간 가격 변동: 용량/요금제 선택 시 통신사별 가격 자동 업데이트
  • 통신사별 가격 한눈에 비교 가능
  • 인터랙티브 가격 선택: 클릭 한 번으로 원하는 옵션 선택
  • 선택 정보 명확 표시: 통신사, 가입유형, 가격이 실시간으로 표시
  • 직관적인 피드백: 선택한 항목은 파란색으로 하이라이트되고 체크마크 표시
  • 상품 이미지로 시각적 인지성 향상
  • 소수점 없는 정수 가격: 55만, 60만 등 명확한 정수로만 표시
  • 간편한 구매 신청 플로우
🎯 선택 옵션 표시 정책
  • 용량 드롭다운: 동일 상품명에 2개 이상의 용량 옵션이 있을 경우에만 표시
  • 요금제 드롭다운: 동일 상품명+용량에 2개 이상의 요금제 옵션이 있을 경우에만 표시
  • 단일 옵션: 옵션이 1개만 있는 경우 드롭다운 없이 고정 텍스트로 표시
  • 예시: "갤럭시 S25 256GB"만 있는 경우 → 용량 선택 없이 "256GB"만 표시
⚠️ 가격 데이터 및 공란 처리
  • DB 기준: 모든 가격은 판매점이 어드민에서 입력한 시세표 DB 데이터를 기준으로 표시
  • 공란 처리: 특정 통신사/가입유형에 가격을 입력하지 않은 경우 "-" 표시
  • 클릭 불가: 가격이 "-"인 항목은 선택 불가 (disabled 처리)
  • 옵션 변경 시: 용량/요금제 변경 시에도 DB에 저장된 가격만 표시되며, 미입력 항목은 "-" 유지
  • 예시: "SKT 공통·번이 = 55만, KT 공통·번이 = 미입력" → SKT는 55만 표시, KT는 "-" 표시

데이터베이스 구조 및 연동

시세표 데이터 구조

중요: 기존 tb_pricetable_phone 테이블 구조를 활용하되, 판매점 시세표 전용 필드를 추가합니다.

핵심 설계 원칙

  • 행 기반 데이터 구조: 1개 상품당 12개의 DB 행으로 저장됩니다
    • 통신사 3개 (SKT, KT, LG U+)
    • × 가입유형 2개 (번호이동, 기기변경)
    • × 지원금 유형 2개 (공시지원금, 선택약정)
    • = 총 12개 행 (3 × 2 × 2 = 12)
  • 상품 정보 자동 연동: 상품 마스터 DB(tb_product_phone)에서 상품명, 용량, 출고가를 자동으로 불러옵니다
  • 가격 표시 방식: 고객에게 보이는 가격은 "할부원금" 필드의 값입니다
  • 판매점별 가격 책정: 각 판매점은 할부원금(단가)만 입력하여 자신만의 시세표를 만듭니다

상품 정보 연동 (tb_product_phone)

판매점이 시세표를 작성할 때, 기존 상품 마스터 DB에서 정보를 불러옵니다.

연동 프로세스

  1. 상품 선택: tb_product_phone에서 상품 목록 조회 
    SELECT product_code, product_name, retail_price
FROM tb_product_phone
WHERE deleted_yn = 'N'
ORDER BY ord;
  2. 용량 선택: 선택한 상품의 용량 옵션 조회 
    SELECT storage_code, capacity
FROM tb_product_phone
WHERE product_group_code = '선택한상품그룹';
  3. 자동 채우기: 출고가(retail_price)는 자동으로 표시
  4. 판매점 입력: 할부원금(단가)만 직접 입력 (만원 단위 정수, 예: 55, 60)
중요: 판매점은 상품을 선택하고 할부원금(단가)만 만원 단위 정수로 입력합니다 (예: 55만원 → "55", -12만원 페이백 → "-12"). 나머지 정보는 자동으로 채워집니다. 음수 입력 시 페이백(고객이 돈을 받고 구매)을 의미합니다.
단가 미입력 시 처리: 할부원금(단가)을 입력하지 않은 상품은 시세표에 포함되지 않습니다. 즉, 해당 상품의 12개 행이 DB에 저장되지 않으며, 고객 화면에도 표시되지 않습니다. 이는 현재 판매하지 않는 상품을 자동으로 제외하는 방식입니다.
가격 입력 및 표시 규칙

📝 어드민 입력 방식

  • 입력 단위: 만원(만₩) 단위 정수로 입력 가능 (양수 또는 음수)
  • 양수 입력 예시 (일반 가격):
    • 55만원 → 55 입력 ✅
    • 60만원 → 60 입력 ✅
    • 100만원 → 100 입력 ✅
  • 음수 입력 예시 (페이백):
    • 12만원 페이백 → -12 입력 ✅
    • 5만원 페이백 → -5 입력 ✅
    • 20만원 페이백 → -20 입력 ✅
  • DB 저장: 입력한 값에 10,000을 곱하여 원 단위로 저장합니다
    • 55 입력 → 550,000원 DB 저장
    • -12 입력 → -120,000원 DB 저장
⚠️ 입력 제한 정책:
  • 정수만 입력 가능: 소수점 값(예: 60.5, 55.5)은 입력할 수 없습니다
  • 양수/음수 허용: 양수는 일반 가격, 음수는 페이백을 의미합니다
  • 잘못된 예시: 60.5 ❌, 55.5 ❌, 48.3 ❌
💰 음수 가격(페이백)이란?

음수 가격은 고객이 상품을 구매하면서 일정 금액을 돌려받는 프로모션입니다. 예를 들어 -12만원은 고객이 휴대폰을 구매하면서 12만원을 현금 또는 포인트로 받는 것을 의미합니다. 이는 매장의 특별 프로모션이나 통신사 리베이트를 반영한 가격입니다.

👁️ 고객 화면 표시 방식

양수 가격 표시:

  • 55 (어드민 입력) → 550,000원 (DB 저장) → "55만" (고객 화면)
  • 60 (어드민 입력) → 600,000원 (DB 저장) → "60만" (고객 화면)
  • 100 (어드민 입력) → 1,000,000원 (DB 저장) → "100만" (고객 화면)

음수 가격 표시 (페이백):

  • -12 (어드민 입력) → -120,000원 (DB 저장) → "12만원 페이백" (고객 화면)
  • -5 (어드민 입력) → -50,000원 (DB 저장) → "5만원 페이백" (고객 화면)
  • -20 (어드민 입력) → -200,000원 (DB 저장) → "20만원 페이백" (고객 화면)
✅ 데이터 흐름:
  • 어드민 입력: 만원 단위 정수로 간편하게 입력 (55, 60, -12 등)
  • DB 저장: 입력값 × 10,000 = 원 단위로 자동 변환하여 저장 (550,000원, 600,000원, -120,000원)
  • 고객 표시:
    • 양수: DB 값 ÷ 10,000 = 만원 단위 (55만, 60만)
    • 음수: |DB 값| ÷ 10,000 = 페이백 표시 (12만원 페이백, 5만원 페이백)
💡 왜 이렇게 설계했나요?

판매점이 만원 단위 정수로 입력하면 훨씬 간편합니다. "550000"을 입력하는 대신 "55"만 입력하면 되니까요. 음수 입력도 지원하여 페이백/리베이트 프로모션을 쉽게 반영할 수 있습니다. 하지만 DB에는 정확한 원 단위로 저장하여 데이터 무결성을 유지합니다. 고객에게는 양수는 가격으로, 음수는 "페이백"으로 읽기 쉽게 표시하여 직관적인 경험을 제공합니다.
