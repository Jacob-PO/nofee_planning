왜 동네성지인가?

복잡한 휴대폰 구매 과정을 단순하게. 믿을 수 있는 동네 매장을 쉽게 찾고,
실시간 가격을 투명하게 비교하며, 진짜 후기로 현명한 선택을 하는 곳.

1

배경 및 문제 정의

현재 상황

휴대폰을 구매하려는 고객들은 어느 매장이 믿을 만한지, 어디가 가격이 저렴한지 알기 어렵습니다. 온라인에서 정보를 찾아도 광고성 글이 많고, 실제 매장을 방문하기 전까지는 정확한 가격을 알 수 없습니다.

고객의 페인 포인트
  • 매장 신뢰도 불투명: 사업자 정보, 인증 여부를 확인하기 어려움
  • 가격 비교 어려움: 통신사별, 요금제별 가격을 일일이 문의해야 함
  • 후기 신뢰도 낮음: 광고성 후기와 진짜 후기를 구분하기 어려움
  • 매장 정보 부족: 운영시간, 위치, 이벤트 정보를 찾기 힘듦
  • 문의 불편함: 전화 문의는 부담스럽고, 답변을 기다려야 함
판매점의 페인 포인트
  • 홍보 채널 부족: 온라인 마케팅 역량이 부족하여 신규 고객 유입 어려움
  • 가격 노출 제한: 오픈마켓에서는 가격을 자유롭게 노출하기 어려움
  • 신뢰 구축 어려움: 오프라인 매장의 신뢰도를 온라인에서 증명하기 어려움
  • 고객 관리 비효율: 문의 대응, 이벤트 홍보 등을 수작업으로 처리

노피의 솔루션

동네성지 스토어는 고객이 내 동네에서 믿을 수 있는 매장을 쉽게 찾고, 투명한 가격으로 비교하며, 진짜 후기로 선택할 수 있도록 돕습니다. 판매점은 인증된 플랫폼에서 자신의 가격과 이벤트를 적극적으로 홍보하고, 고객과 직접 소통할 수 있습니다.

핵심 기능

2.1 매장 탐색 및 검색

지역 기반 필터링

고객은 자신의 동네에서 가까운 매장을 쉽게 찾을 수 있습니다.

  • 2단계 지역 필터: 시/도 선택 → 구/군 선택
  • 지원 지역: 서울, 부산, 경기, 인천, 대구, 대전, 광주, 울산, 제주 (9개 시/도)
  • 동적 옵션 업데이트: 시/도 선택 시 해당 구/군 목록 자동 표시
  • 검색창 연동: 선택한 지역이 검색창에 표시 ("서울 강남구")
📍 향후 개선: GPS 기반 자동 위치 인식, 반경(km) 내 검색, 지도 뷰 제공
매장 리스트 및 정보

각 매장의 핵심 정보를 한눈에 비교할 수 있습니다.

  • 매장 썸네일: 매장 대표 이미지
  • 평점 및 후기 수: 별점 (5.0점 만점) + 후기 개수
  • 위치 정보: "서울 은평구" 형태로 표시
  • 방문 수: 누적 방문자 수 (신뢰도 지표)
  • 매장 설명: 해시태그 또는 짧은 소개 문구
  • 더보기/접기: 초기 5개 표시, "더보기" 클릭 시 전체 표시

📋 홈 탭 - 인증 정보

신뢰도 강화

고객이 매장을 믿고 선택할 수 있도록 인증 정보를 투명하게 공개합니다.

  • 사업자 정보 인증완료 배지
  • 인증 항목: 사업자등록증 확인, 승낙서 확인, 본인 인증
  • 상세 정보 토글: "자세히 보기" 클릭 시 사업자번호, 통신판매업 신고번호, 인증일자 표시
✅ 토스의 철학: "복잡한 것을 단순하게" - 중요한 정보는 한눈에, 상세 정보는 클릭 한 번으로.

📰 소식 탭 - 이벤트 피드

실시간 이벤트 홍보

판매점이 직접 등록한 이벤트를 고객에게 실시간으로 전달합니다.

  • 피드 구조: 매장 프로필 + 날짜 + 제목 + 내용 + 제품 카드
  • 제품 카드 정보: 제품 이미지, 이름, 통신사, 용량, 개통방식, 할인방식, 요금제, 할부원금/실제 구매가
  • 할인율 배지: "158%" 같은 눈에 띄는 할인율 표시
  • CTA 버튼: "이벤트 참여하기" (그라데이션 배경, 클릭 시 문의/구매 프로세스)

⭐ 후기 탭 - 고객 후기

진짜 후기, 진짜 신뢰

실제 고객의 경험을 바탕으로 매장을 선택할 수 있습니다.

  • 후기 통계: 평균 평점 (별 아이콘) + 총 후기 개수
  • 후기 작성 유도: "후기 작성하기" 버튼 (underline 스타일, 부담 없는 UI)
  • 후기 카드: 작성자명 (마스킹), 별점, 작성일, 내용 (2줄 제한), 사진 (선택)
  • 더보기 기능: 초기 5개 표시, 5개씩 추가 로드
📝 향후 개선: 사진 후기 우선 노출, 도움이 돼요 투표, 판매점 댓글 기능

2.3 후기 작성 시스템

쉽고 빠른 후기 작성

고객이 10초 만에 후기를 남길 수 있도록 단순하고 직관적으로 설계했습니다.

입력 폼 구조

  1. 별점 선택 (필수)
    • 1-5개의 큰 별 버튼 (터치하기 쉬운 크기)
    • 시각적 피드백: 선택된 별 = 노란색, 호버 시 scale(1.1)
    • 텍스트 피드백: 1점 = "별로예요", 5점 = "최고예요!"
  2. 제목 입력 (필수)
    • 플레이스홀더: "후기 제목을 입력해주세요"
    • 단일 줄 입력
  3. 상세 내용 (필수, 최소 10자)
    • 플레이스홀더: "매장 이용 경험을 자세히 작성해주세요"
    • 6줄 텍스트 영역, 최대 500자
    • 실시간 글자 수 카운트 표시
  4. 사진 첨부 (선택)
    • 점선 테두리 업로드 영역 ("+" 아이콘)
    • 이미지 미리보기 (최대 320px 너비)
    • 삭제 버튼 제공

유효성 검사

  • 별점 미선택 시: "별점을 선택해주세요" 알림
  • 제목 미입력 시: "제목을 입력해주세요" 알림 + 포커스 이동
  • 내용 10자 미만 시: "내용을 10자 이상 입력해주세요" 알림 + 포커스 이동
  • 제출 성공 시: "후기가 작성되었습니다!" 알림 → 폼 초기화 → 매장 상세 (후기 탭)으로 자동 이동
✅ UX 포인트: 필수 항목은 명확히, 선택 항목은 부담 없이. 실패 시 즉각 피드백, 성공 시 매끄러운 전환.

2.4 단골 및 문의 기능

단골 등록
  • "단골 할래요" 버튼 (초기 상태)
  • 클릭 시 "단골이에요"로 변경
  • 토글 방식 (다시 클릭 시 해제)
  • 향후: 단골 매장 알림, 맞춤 추천
문의하기
  • "문의하기" 버튼 클릭
  • 바텀시트 표시 (슬라이드 업 애니메이션)
  • 확인 아이콘 + 안내 메시지
  • 버튼 상태 변경: "문의 취소하기"
  • 취소 클릭 시 원래 상태로 복귀

2.5 이벤트 피드 (메인 페이지)

현재 진행중인 이벤트 섹션

메인 페이지 하단에 전체 매장의 이벤트를 모아서 보여줍니다.

  • 섹션 제목: "현재 진행중인 이벤트"
  • 초기 노출: 3개의 이벤트 카드
  • 더보기/접기: 버튼 클릭 시 모든 이벤트 표시/숨김
  • 카드 클릭: 해당 매장의 상세 페이지로 이동
  • 제품 정보: 이미지, 이름, 통신사, 용량, 개통방식, 할인방식, 요금제, 가격

사용자 플로우

3.1 고객 - 매장 탐색 및 선택

1 메인 페이지 진입
고객이 노피 앱/웹에서 "동네성지" 메뉴를 선택
↓
2 지역 필터 선택 (선택)
검색창 클릭 → 필터 드롭다운 표시 → 시/도 선택 → 구/군 선택 → "적용" 버튼 클릭
결과: 선택한 지역이 검색창에 표시되고, 매장 리스트가 필터링됨
↓
3 매장 리스트 확인
평점, 후기 수, 위치, 방문 수 등을 비교하며 매장 선택
"더보기" 버튼 클릭 시 추가 매장 확인 가능
↓
4 매장 카드 클릭
매장 상세 페이지로 이동 (홈 탭이 기본으로 표시됨)
↓
5 매장 정보 확인 (홈 탭)
매장 소개, 인증 정보, 이벤트, 운영시간, 위치 등을 확인
"자세히 보기" 클릭 시 상세 정보 토글
↓
6 소식 탭 확인 (선택)
매장의 최신 이벤트/소식 피드 확인
제품 카드의 "이벤트 참여하기" 버튼 클릭 가능
↓
7 후기 탭 확인
다른 고객의 후기를 확인하여 최종 의사결정
"더보기" 클릭 시 추가 후기 5개씩 로드
↓
8 액션 선택
A. "단골 할래요" 클릭 → 단골 등록
B. "문의하기" 클릭 → 바텀시트 표시 → 확인 → 문의 접수
C. "뒤로가기" 클릭 → 메인 리스트로 복귀

3.2 고객 - 후기 작성

1 매장 상세 페이지 → 후기 탭
↓
2 "후기 작성하기" 버튼 클릭
후기 작성 페이지로 전환 (슬라이드 애니메이션)
↓
3 별점 선택 (필수)
1-5개의 별 중 하나 클릭 → 텍스트 피드백 표시 ("최고예요!" 등)
↓
4 제목 및 내용 입력 (필수)
제목: 한 줄 입력
내용: 6줄 텍스트 영역 (최소 10자, 최대 500자)
실시간 글자 수 카운트 표시
↓
5 사진 첨부 (선택)
"+" 버튼 클릭 → 파일 선택 → 이미지 미리보기
삭제 버튼으로 취소 가능
↓
6 "작성 완료" 버튼 클릭
유효성 검사 실행 (별점, 제목, 내용 확인)
↓
7 제출 성공
"후기가 작성되었습니다!" 알림
폼 초기화 → 매장 상세 페이지 (후기 탭)으로 자동 이동
💡 UX 인사이트: 유효성 검사 실패 시 해당 필드로 자동 포커스 이동, 에러 메시지는 명확하고 친절하게.

데이터 구조

4.1 매장 데이터 (storesData)

필드명 타입 설명
id String 매장 고유 ID (예: "store-1")
name String 매장명 (예: "번개폰 은평점")
image String (URL) 매장 썸네일 이미지 URL
rating String 평점 (예: "5.0", "-"는 평점 없음)
reviewCount Number 후기 개수
location String 위치 (예: "서울 은평구")
visitCount Number 누적 방문 수
description String 매장 설명 (해시태그 또는 문구)
hidden Boolean 더보기 이전 숨김 여부 (true/false)

4.2 이벤트 데이터 (eventsData)

필드명 타입 설명
id String 이벤트 고유 ID
storeName String 매장명
storeImage String (URL) 매장 프로필 이미지
date String 작성일 (예: "2025. 09. 07")
title String 이벤트 제목
product Object 제품 정보 객체 (하위 필드 참고)
product.name String 제품명 (예: "갤럭시 Z 폴드 7")
product.image String (URL) 제품 이미지 URL
product.carrier String 통신사 (SKT, KT, LG U+)
product.storage String 저장용량 (예: "256GB")
product.activationType String 개통방식 (번호이동, 기기변경)
product.discountType String 할인방식 (선택약정, 공시지원금)
product.monthlyFee String 월 요금 (예: "109,000원/월")
product.price String 할부원금/실제 구매가 (예: "1,174,300원")
product.priceLabel String 가격 라벨 (할부원금, 실제 구매가)
product.discountRate String (선택) 할인율 배지 (예: "158%")

4.3 후기 데이터 (reviewsData)

필드명 타입 설명
storeId String 매장 ID (외래키)

reviewerName String 작성자명 (마스킹, 예: "김*수")

content String 후기 본문

기술 스택 및 구현

5.1 프론트엔드

현재 프로토타입
  • HTML5: 시맨틱 마크업
  • Tailwind CSS: 유틸리티 기반 스타일링 (CDN)
  • Vanilla JavaScript: 라이브러리 없는 순수 JS
  • SVG 아이콘: 인라인 SVG로 벡터 아이콘 구현
실제 서비스화를 위한 스택
  • React 또는 Vue.js: 컴포넌트 기반 UI 프레임워크
  • TypeScript: 타입 안정성 확보
  • Styled-components 또는 Emotion: CSS-in-JS
  • React Query (TanStack Query): 서버 상태 관리
  • Zustand 또는 Recoil: 클라이언트 상태 관리
  • Next.js: SSR/SSG, SEO 최적화

5.2 백엔드 (필요한 API)

API 엔드포인트 메서드 설명
GET /api/stores GET 매장 리스트 조회 (필터: 지역, 정렬)
GET /api/stores/:id GET 매장 상세 정보 조회
GET /api/stores/:id/feeds GET 매장의 이벤트/소식 피드 조회
GET /api/stores/:id/reviews GET 매장 후기 목록 조회 (페이징)
POST /api/reviews POST 후기 작성 (별점, 제목, 내용, 이미지)
POST /api/subscriptions POST 단골 등록
DELETE /api/subscriptions/:id DELETE 단골 해제
POST /api/inquiries POST 문의 접수
GET /api/events GET 전체 이벤트 피드 조회 (메인 페이지)

5.3 데이터베이스

주요 테이블
  1. stores: 매장 정보 (ID, 이름, 이미지, 위치, 소개, 인증 정보, 운영시간 등)
  2. feeds: 매장 피드/이벤트 (ID, store_id, 제목, 내용, 제품 정보, 작성일)
  3. reviews: 고객 후기 (ID, store_id, user_id, 별점, 제목, 내용, 이미지, 작성일)
  4. subscriptions: 단골 관계 (ID, user_id, store_id, 등록일)
  5. inquiries: 문의 내역 (ID, user_id, store_id, 상태, 작성일)

8.1 고객 관점

  • 신뢰할 수 있는 매장 발견: 인증 정보, 후기로 안심하고 선택
  • 투명한 가격 비교: 통신사별, 요금제별 가격을 한눈에 확인
  • 편리한 문의 프로세스: 전화 부담 없이 온라인으로 간편하게 문의
  • 실시간 이벤트 확인: 놓치지 않는 최신 할인 정보
  • 동네 매장 지원: 대형 온라인몰이 아닌 내 동네 매장을 선택하는 뿌듯함

8.2 판매점 관점

  • 신규 고객 유입 증가: 온라인 홍보 채널 확보
  • 가격 경쟁력 어필: 자유롭게 가격을 노출하고 홍보
  • 신뢰도 강화: 인증 배지, 후기로 신뢰 구축
  • 고객 관리 효율화: 단골, 문의, 이벤트를 한 곳에서 관리
  • 브랜드 이미지 향상: 전문적이고 투명한 이미지 구축

8.3 노피 플랫폼 관점

  • 사용자 체류 시간 증가: 매장 탐색, 후기 작성 등 인터랙션 증가
  • 네트워크 효과: 매장↔고객 연결로 플랫폼 가치 상승
  • 데이터 축적: 후기, 방문 데이터로 추천 알고리즘 고도화
  • 수익 모델 확장: 프리미엄 매장 등록, 광고, 이벤트 프로모션 등
  • 브랜드 차별화: "투명하고 믿을 수 있는 플랫폼" 이미지 강화

동네성지는 단순한 매장 리스트가 아닙니다
